# AI Innovation Challenge 2026 - 데이터베이스 실행 가이드

## 📋 전체 구조 요약

### 데이터베이스 테이블 (총 6개)

```
1. brands               브랜드 정보 (초기 5개 브랜드 자동 삽입)
2. products             상품 정보 (빈 상태 - 크롤링 데이터 임포트 필요)
3. personas             페르소나 정보 (빈 상태 - 프론트엔드에서 추가)
4. product_personas     상품-페르소나 매핑 (추천 시스템용)
5. persona_analysis_results  페르소나 분석 결과
6. persona_solutions    페르소나 솔루션
```

### 핵심 파일 구조

```
database/
├── docker-compose.yml          # Docker 설정
├── .env                        # 환경변수 설정
├── database.py                 # DB 연결 설정
├── models.py                   # SQLAlchemy ORM 모델
├── api_server.py               # FastAPI 서버
├── test_connection.py          # DB 연결 테스트
├── jsonl_to_sql.py             # JSONL → SQL 변환 도구
│
└── init/                       # SQL 초기화 스크립트
    ├── 01-create-tables.sql    # 테이블 생성
    ├── 02-seed-initial-data.sql # 브랜드 초기 데이터
    ├── 04-insert-brands.sql    # (자동생성) 추가 브랜드
    ├── 05-insert-products.sql  # (자동생성) 상품 데이터
    └── 06-insert-product-personas.sql  # (자동생성) 매핑 데이터
```

## 📊 데이터 확인 쿼리

### pgAdmin 또는 SQL 클라이언트에서 실행

```sql
-- 1. 브랜드별 상품 수
SELECT
    b.name AS brand_name,
    COUNT(p.id) AS product_count
FROM brands b
LEFT JOIN products p ON b.id = p.brand_id
GROUP BY b.name
ORDER BY product_count DESC;

-- 2. 페르소나 목록
SELECT
    persona_key,
    name,
    gender,
    age_group,
    skin_types,
    skin_concerns
FROM personas
ORDER BY created_at DESC;

-- 3. 페르소나별 추천 상품 수
SELECT
    p.name AS persona_name,
    COUNT(pp.id) AS recommended_products
FROM personas p
LEFT JOIN product_personas pp ON p.id = pp.persona_id
GROUP BY p.name
ORDER BY recommended_products DESC;

-- 4. 특정 페르소나의 추천 상품 (상위 10개)
SELECT
    pr.product_name,
    b.name AS brand_name,
    pr.category,
    pr.price,
    pp.relevance_score
FROM product_personas pp
JOIN personas p ON pp.persona_id = p.id
JOIN products pr ON pp.product_id = pr.id
JOIN brands b ON pr.brand_id = b.id
WHERE p.persona_key = 'user_001'
ORDER BY pp.relevance_score DESC
LIMIT 10;
```

## 🐛 문제 해결

### 1. 포트 충돌 문제

**증상:** `port is already allocated`

**해결:**
```bash
# 실행 중인 컨테이너 확인
docker ps

# 충돌하는 컨테이너 중지
docker stop <container_id>

# 또는 .env 파일에서 포트 변경
POSTGRES_PORT=5433
PGADMIN_PORT=5051
```

### 2. 데이터베이스 연결 실패

**증상:** `Connection refused`

**해결:**
```bash
# 컨테이너 상태 확인
docker-compose ps

# 로그 확인
docker-compose logs postgres

# 재시작
docker-compose restart postgres
```

### 3. SQL 파일 임포트 실패

**증상:** `No such file or directory`

**해결:**
```bash
# 파일 존재 확인
ls -la init/

# JSONL 변환 다시 실행
python jsonl_to_sql.py

# Windows에서는 PowerShell 사용
Get-Content init\05-insert-products.sql | docker exec -i ai-innovation-postgres psql -U postgres ai_innovation_db
```

### 4. API 서버 실행 실패

**증상:** `ModuleNotFoundError`

**해결:**
```bash
# 필요한 패키지 설치
pip install -r requirements.txt

# 환경 확인
python --version
pip list
```

## 📁 중요 파일 설명

### database.py
- PostgreSQL 연결 설정
- SQLAlchemy 엔진 및 세션 관리
- `get_db()` 함수: FastAPI Depends용

### models.py
- 6개 테이블의 SQLAlchemy ORM 모델
- Brand, Product, Persona, ProductPersona, PersonaAnalysisResult, PersonaSolution

### api_server.py
- FastAPI 기반 REST API 서버
- CORS 설정 포함
- Pydantic 모델로 요청/응답 검증

### jsonl_to_sql.py
- 크롤링 데이터(JSONL) → SQL INSERT 문 변환
- `생성된_문서` → `generated_document` 매핑
- 배치 INSERT로 성능 최적화

### test_connection.py
- 데이터베이스 연결 테스트
- 테이블 생성 확인
- 기본 데이터 조회 테스트
