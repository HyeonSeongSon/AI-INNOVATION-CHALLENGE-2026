### 현재 테이블 구조 (수정 완료):

```sql
CREATE TABLE products (
    -- 기본키
    id SERIAL PRIMARY KEY,
    brand_id INTEGER REFERENCES brands(id) ON DELETE SET NULL,

    -- 벡터DB 연동
    vector_db_id VARCHAR(200),           -- 벡터db id
    indexing_number VARCHAR(200),         -- 인덱싱 번호

    -- 기본 상품 정보
    product_code VARCHAR(100),
    product_name VARCHAR(500) NOT NULL,   -- 상품명
    category VARCHAR(100),
    sub_category VARCHAR(100),

    -- 가격 정보
    original_price DECIMAL(10, 2),        -- 원가
    discount_rate DECIMAL(5, 2),          -- 할인율 (%)
    sale_price DECIMAL(10, 2),            -- 판매가

    -- 평점/리뷰
    rating DECIMAL(3, 2),                 -- 별점 (0.0 ~ 5.0)
    review_count INTEGER DEFAULT 0,       -- 리뷰 개수

    -- 페르소나 매칭용 속성
    skin_types TEXT[] DEFAULT '{}',                -- 피부타입 (배열)
    personal_colors TEXT[] DEFAULT '{}',           -- 퍼스널 컬러 (배열)
    base_shades TEXT[] DEFAULT '{}',               -- 베이스 호수 (배열)
    concern_keywords TEXT[] DEFAULT '{}',          -- 고민 키워드 (배열)
    makeup_colors TEXT[] DEFAULT '{}',             -- 메이크업 선호 색상 (배열)
    preferred_ingredients TEXT[] DEFAULT '{}',     -- 선호 성분 (배열)
    avoided_ingredients TEXT[] DEFAULT '{}',       -- 기피 성분 (배열)
    preferred_scents TEXT[] DEFAULT '{}',          -- 선호 향 (배열)
    values_keywords TEXT[] DEFAULT '{}',           -- 가치관 (배열)
    dedicated_products TEXT[] DEFAULT '{}',        -- 전용제품 (배열)

    -- URL 및 이미지
    product_url TEXT,                     -- 판매 URL
    image_urls TEXT[] DEFAULT '{}',       -- 상품 이미지 URL 배열

    -- 상세 정보
    description TEXT,
    generated_document TEXT,              -- GPT 생성 문서

    -- 상품 태그 (기존 JSONB)
    tags JSONB DEFAULT '{}',
    buyer_statistics JSONB DEFAULT '{}',
    detailed_info JSONB DEFAULT '{}',

    -- 타임스탬프
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,

    UNIQUE(brand_id, product_code)
);
```

### 주요 인덱스:
```sql
-- 기본 인덱스
CREATE INDEX idx_products_brand_id ON products(brand_id);
CREATE INDEX idx_products_category ON products(category);

CREATE INDEX idx_products_preferred_ingredients ON products USING GIN(preferred_ingredients);
CREATE INDEX idx_products_tags ON products USING GIN(tags);
```

## 3. 페르소나 정보 테이블 (personas)

### persona_categories.json 기반 구조:

```sql
CREATE TABLE personas (
    id SERIAL PRIMARY KEY,
    persona_key VARCHAR(100) NOT NULL UNIQUE,
    name VARCHAR(200) NOT NULL,
    description TEXT,

    -- 1. 기본 정보 (basic_info)
    gender VARCHAR(20),                    -- 성별
    age_group VARCHAR(50),                 -- 연령대

    -- 2. 피부 스펙 (skin_spec)
    skin_types TEXT[] DEFAULT '{}',        -- 피부타입 (건성, 지성, 복합성, 민감성 등)
    personal_color VARCHAR(50),            -- 퍼스널컬러
    base_shade VARCHAR(10),                -- 베이스호수

    -- 3. 피부 고민 (skin_concerns) - 최대 3개
    skin_concerns TEXT[] DEFAULT '{}',

    -- 4. 메이크업 선호 (makeup_preference)
    preferred_point_colors TEXT[] DEFAULT '{}',

    -- 5. 성분 선호 (ingredient_preference)
    preferred_ingredients TEXT[] DEFAULT '{}',
    avoided_ingredients TEXT[] DEFAULT '{}',
    preferred_scents TEXT[] DEFAULT '{}',

    -- 6. 가치관 (values)
    special_conditions TEXT[] DEFAULT '{}',

    -- 추가 정보
    budget_range VARCHAR(50),

    -- 메타데이터
    metadata JSONB DEFAULT '{}',

    -- 타임스탬프
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

## 4. 상품-페르소나 매핑 테이블 (product_personas)

**목적:** 페르소나와 상품 간의 추천 매칭 저장

```sql
CREATE TABLE product_personas (
    id SERIAL PRIMARY KEY,
    product_id INTEGER NOT NULL REFERENCES products(id) ON DELETE CASCADE,
    persona_id INTEGER NOT NULL REFERENCES personas(id) ON DELETE CASCADE,
    relevance_score DECIMAL(5, 4) CHECK (relevance_score >= 0 AND relevance_score <= 1),
    matched_attributes JSONB DEFAULT '{}',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(product_id, persona_id)
);
```

**현재 데이터:** 0개

**필드 설명:**
- `relevance_score`: 상품과 페르소나의 관련성 점수 (0.0 ~ 1.0)
- `matched_attributes`: 매칭된 속성 정보 (JSONB)
  ```json
  {
    "matched_skin_types": ["건성", "민감성"],
    "matched_concerns": ["보습", "진정"],
    "matched_ingredients": ["히알루론산", "세라마이드"],
    "score_breakdown": {
      "skin_type_match": 0.95,
      "concern_match": 0.88,
      "ingredient_match": 0.92
    }
  }
  ```

## 5. 페르소나 분석 결과 테이블 (persona_analysis_results)

**목적:** 페르소나에 대한 AI 분석 결과 저장

```sql
CREATE TABLE persona_analysis_results (
    id SERIAL PRIMARY KEY,
    persona_id INTEGER NOT NULL REFERENCES personas(id) ON DELETE CASCADE,
    product_id INTEGER REFERENCES products(id) ON DELETE SET NULL,
    analysis_type VARCHAR(50) NOT NULL,           -- 분석 유형
    analysis_result JSONB DEFAULT '{}',            -- 분석 결과 (JSON)
    confidence_score DECIMAL(5, 4),                -- 신뢰도 점수
    generated_message TEXT,                        -- 생성된 메시지
    metadata JSONB DEFAULT '{}',
    analyzed_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

**분석 유형 예시:**
- `skin_analysis`: 피부 분석
- `product_recommendation`: 상품 추천
- `routine_suggestion`: 루틴 제안
- `ingredient_compatibility`: 성분 호환성 분석

**분석 결과 예시:**
```json
{
  "skin_condition": {
    "hydration_level": "low",
    "sensitivity": "high",
    "primary_concerns": ["건조", "각질"]
  },
  "recommendations": {
    "morning_routine": [...],
    "evening_routine": [...]
  },
  "key_ingredients": ["히알루론산", "세라마이드", "판테놀"]
}
```

## 6. 페르소나 솔루션 테이블 (persona_solutions)

**목적:** 분석 결과 기반 솔루션 제안

```sql
CREATE TABLE persona_solutions (
    id SERIAL PRIMARY KEY,
    analysis_result_id INTEGER NOT NULL REFERENCES persona_analysis_results(id) ON DELETE CASCADE,
    solution_type VARCHAR(50) NOT NULL,            -- 솔루션 유형
    title VARCHAR(500) NOT NULL,                   -- 솔루션 제목
    description TEXT,                              -- 솔루션 설명
    recommended_products JSONB DEFAULT '[]',       -- 추천 상품 목록
    priority INTEGER DEFAULT 0,                    -- 우선순위
    effectiveness_score DECIMAL(5, 4),             -- 효과 점수
    implementation_guide TEXT,                     -- 실행 가이드
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

## 💡 사용 예시

### 1. 상품 검색 (피부타입 기반)

```sql
SELECT product_name, brand_name, sale_price, rating
FROM products p
JOIN brands b ON p.brand_id = b.id
WHERE '건성' = ANY(skin_types)
  AND '보습' = ANY(concern_keywords)
ORDER BY rating DESC, review_count DESC
LIMIT 10;
```

### 2. 페르소나 매칭

```sql
SELECT
    p.product_name,
    pp.relevance_score,
    pp.matched_attributes
FROM product_personas pp
JOIN products p ON pp.product_id = p.id
WHERE pp.persona_id = (SELECT id FROM personas WHERE persona_key = 'user_001')
ORDER BY pp.relevance_score DESC
LIMIT 20;
```

### 3. 가격대별 추천 상품

```sql
SELECT
    product_name,
    sale_price,
    discount_rate,
    rating,
    review_count
FROM products
WHERE sale_price BETWEEN 10000 AND 50000
  AND '히알루론산' = ANY(preferred_ingredients)
ORDER BY
    rating DESC,
    review_count DESC
LIMIT 10;
```
